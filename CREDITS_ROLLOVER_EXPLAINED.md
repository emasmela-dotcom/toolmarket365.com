# 💳 Credits Rollover - Clear Explanation

## 🎯 **The Rule: Free Credits Reset, Purchased Credits Roll Over**

### **Free Monthly Credits (25/month):**
```
✅ Reset every month
❌ Don't accumulate
❌ Don't roll over
❌ Use them or lose them
```

### **Purchased Credits:**
```
✅ Roll over month to month
✅ Accumulate
✅ Don't expire (or expire after 6 months)
✅ Use them anytime
```

---

## 📊 **How It Works - Examples**

### **Month 1:**
```
User on Starter plan:
- Gets 25 free credits (new month)
- Buys 100 credits for $10
- Uses 20 credits this month
- Has 5 free credits left (will reset)
- Has 80 purchased credits left (will roll over)
```

### **Month 2:**
```
User on Starter plan:
- Gets 25 NEW free credits (reset from last month)
- Still has 80 purchased credits (rolled over from last month)
- Total: 105 credits available
- Uses 30 credits this month
- Has 0 free credits left (will reset)
- Has 75 purchased credits left (will roll over)
```

### **Month 3:**
```
User on Starter plan:
- Gets 25 NEW free credits (reset from last month)
- Still has 75 purchased credits (rolled over from last month)
- Total: 100 credits available
- Uses 10 credits this month
- Has 15 free credits left (will reset - lost!)
- Has 90 purchased credits left (will roll over)
```

---

## 💡 **Why This Makes Sense**

### **1. Free Credits Reset Monthly:**
**Why:**
- ✅ Encourages monthly engagement
- ✅ Prevents stockpiling free credits
- ✅ Users use them or lose them
- ✅ Creates monthly habit

**Example:**
```
User doesn't use free credits this month
Next month: Gets fresh 25 credits
Old free credits: Gone (reset)
```

### **2. Purchased Credits Roll Over:**
**Why:**
- ✅ User paid for them - they should keep them
- ✅ Encourages larger purchases
- ✅ Creates commitment
- ✅ Better user experience

**Example:**
```
User buys 100 credits for $10
Uses 20 this month
Next month: Still has 80 credits
They keep rolling over
```
